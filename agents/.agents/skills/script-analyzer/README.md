# Script Analyzer Skill

安全分析脚本的功能、影响范围和潜在风险，避免盲目运行从网上下载的脚本。

## 功能特性

- **功能分析**: 识别脚本的主要用途和操作
- **影响范围**: 检测文件系统、网络和系统级修改
- **风险评估**: 评估安全风险等级（低/中/高）
- **建议生成**: 提供安全执行建议

## 使用方法

### 1. 使用分析脚本

```bash
# 基本分析
python3 scripts/analyze.py /path/to/script.sh

# JSON格式输出
python3 scripts/analyze.py --json /path/to/script.sh

# 详细分析
python3 scripts/analyze.py --full /path/to/script.sh
```

### 2. 使用LLM分析

直接对脚本内容进行智能分析，提供更详细的解释和建议。

## 分析维度

### 功能分析

- 脚本用途（安装、配置、构建等）
- 主要操作类型
- 依赖的工具和库

### 影响范围

- 文件操作（创建、修改、删除）
- 网络活动（下载、上传、API调用）
- 系统修改（服务、配置、权限）

### 风险评估

- 安全风险（凭证泄露、权限提升）
- 稳定性风险（系统配置更改）
- 隐私风险（数据收集、追踪）

## 示例

### 安全脚本示例

```bash
# examples/safe_script.sh
# 仅显示系统信息，无风险操作
```

### 高风险脚本示例

```bash
# examples/risky_script.sh
# 包含远程执行、系统修改等高风险操作
```

## 最佳实践

1. **运行前分析**: 始终在运行未知脚本前进行分析
2. **隔离环境**: 在VM、容器或沙箱中运行高风险脚本
3. **备份数据**: 运行系统修改脚本前备份重要数据
4. **监控执行**: 使用系统监控工具观察脚本行为
5. **验证来源**: 检查脚本来源和作者信誉

## 注意事项

- 静态分析无法检测所有动态行为
- 高度混淆的脚本可能难以分析
- 外部依赖的行为可能随时间变化
- 新型攻击模式可能无法识别
